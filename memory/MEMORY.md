# Memory Index

Durable repo-level memory. Each entry is one line pointing to a file
in this directory:

`- [Title](file.md) — one-line hook under ~150 chars.`

## Memory Types

Each memory lives in its own file next to this index. Types:

- **user** — who the user is, role, preferences, responsibilities.
- **feedback** — guidance the user has given about how to work here;
  include **Why:** and **How to apply:** lines.
- **project** — ongoing work, decisions, constraints, deadlines;
  include **Why:** and **How to apply:** lines.
- **reference** — pointers to external systems (Linear projects,
  dashboards, Slack channels, external docs).

See `~/.claude/CLAUDE.md` (auto memory section) for the full protocol
on what to save, what not to save, and how entries are structured.
